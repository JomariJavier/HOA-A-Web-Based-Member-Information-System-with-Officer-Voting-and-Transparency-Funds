====================================================================
PERSONAL INFORMATION SYSTEM (PIS) — Plans & References
====================================================================

WHAT IS THIS MODULE?
--------------------------------------------------------------------
The PIS is basically a digital notebook for ALL the homeowners in
the association. Instead of paper lists that can get lost, everything
is stored in the database and shown on the website.

Think of it like the school's student records office — but for
homeowners. The Admin (officer) is the clerk, and the Members are
the students. Students can't peek at the records, only the clerk can.

WHO CAN USE IT?
--------------------------------------------------------------------
    Admin    → FULL access (view, add, edit, delete members)
    Member   → NO access to this module at all
    Superadmin → NO access (this is the Admin's job)

    ├── MemberController.java← The API endpoints (GET, POST)
    ├── MemberRepository.java← Talks to the database
    └── MemberService.java   ← EMPTY placeholder (business logic layer)

DATABASE TABLE: "members"
    Columns: id, full_name, birth_date, hoa_address, date_registered,
             marital_status, gender, family_members, role, status


====================================================================
API ENDPOINTS (Currently Live)
====================================================================

    METHOD   URL                     WHAT IT DOES
    ──────   ──────────────────────  ─────────────────────────────
    GET      /api/members            Returns ALL members as JSON
    GET      /api/members/{id}       Returns ONE member by ID
    POST     /api/members            Creates a new member record

    Missing endpoints (not built yet):
    PUT      /api/members/{id}       Update an existing member
    DELETE   /api/members/{id}       Delete / deactivate a member
