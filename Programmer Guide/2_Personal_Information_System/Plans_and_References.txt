====================================================================
PERSONAL INFORMATION SYSTEM (PIS) — Plans & References
====================================================================

REFERENCE: intergrationplan.md -> Phase 2: Personal Information System

Objective: Digital management of HOA member records, residency details, and membership status. 
Restricts access to authorized Officers (ADMIN) to protect resident privacy.

Tech Stack:
- Backend: Spring Boot, PostgreSQL
- Frontend: React (Vite), Material You Design

Entity: Member
- id: Long (Identity)
- fullName: String
- birthDate: LocalDate
- hoaAddress: String
- dateRegistered: LocalDate (Auto-set)
- email: String
- contactNumber: String
- status: ENUM ('Active', 'Inactive', 'Suspended')
- role: ENUM ('Member', 'Officer')

Key Views (Components):
1. Member Directory (MemberDirectory.jsx): Searchable table of all members.
2. Member Profile (MemberProfile.jsx): Detailed view of a single member's records.
3. Member Registration (MemberRegistration.jsx): Form for onboarding new residents.

Integration Status:
[PROGRESS] Basic Member entity exists.
[TODO] Update Member entity with email and contactNumber fields.

Access Rules:
- ADMIN: Full access to all PIS views and endpoints.
- MEMBER: No access (Access Denied state).
