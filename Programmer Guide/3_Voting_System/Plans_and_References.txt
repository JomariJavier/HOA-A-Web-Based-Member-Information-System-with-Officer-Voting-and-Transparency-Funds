VOTING SUBSYSTEM (HOA VOTING ROOM) — Plans & References
====================================================================

REFERENCE: intergrationplan.md -> Phase 3: Voting System

Objective: Secure, transparent, and anonymous digital elections. 
Replaces paper ballots with a system that prevents double voting while protecting voter privacy.

Tech Stack:
- Backend: Spring Boot, PostgreSQL
- Frontend: React (Vite), Material You Design

Core Entities:
1. Election: Represents a specific poll (e.g., "Presidential Election 2026").
2. Candidate: Represents a person running in an election.
3. VoteTracker: Tracks WHO has voted (to prevent double voting).
4. Vote: Stores the actual vote (decoupled from the user for anonymity).

Key Views (Components):
- VotingRoom.jsx: The central hub for all voting activities.
- ElectionList.jsx: Displays active and past elections.
- VotingBallot.jsx: The interface for casting a vote.
- ElectionManagement.jsx: Admin tool for creating new elections.
