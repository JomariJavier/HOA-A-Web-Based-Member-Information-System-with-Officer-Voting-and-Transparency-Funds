====================================================================
    PROGRAMMER GUIDE — READ THIS FIRST!
====================================================================

Hey team! 👋

Welcome to the Programmer Guide. Think of this folder as our
project's "instruction manual" — the place where we keep track
of WHAT we're building, WHAT's left to do, and WHAT went wrong.

If you've ever used a recipe book, this is basically that, but
for code.


====================================================================
    WHAT'S INSIDE?
====================================================================

This folder has one sub-folder for each MODULE (or "feature") of
our HOA system. Here's a quick map:

    Programmer Guide/
    │
    ├── +README.txt          ← You are here! Start here always.
    │
    ├── 1_RBAC_Login_System/        ← The login page & user accounts
    ├── 2_Personal_Information_System/  ← Member records management
    ├── 3_Voting_System/            ← Anonymous officer elections
    ├── 4_Auditing_System/          ← Money tracking & transparency
    ├── 5_Public_Relations_System/  ← Announcements & complaints
    └── 6_Audit_Dashboard/          ← The main summary screen


====================================================================
    WHAT'S INSIDE EACH MODULE FOLDER?
====================================================================

Every module folder has the SAME set of files. Think of each file
like a different "notebook" for that module:

    ┌─────────────────────────────┬──────────────────────────────────┐
    │ File                        │ What It's For                    │
    ├─────────────────────────────┼──────────────────────────────────┤
    │ Plans_and_References.txt    │ The "blueprint".                 │
    │                             │ How the module is supposed to    │
    │                             │ work, what screens it has, and   │
    │                             │ what we plan to add later.       │
    ├─────────────────────────────┼──────────────────────────────────┤
    │ Todo.txt                    │ The "shopping list".             │
    │                             │ Tasks that still need to be      │
    │                             │ done. Check here to see what     │
    │                             │ you can pick up and work on.     │
    ├─────────────────────────────┼──────────────────────────────────┤
    │ Changelog.txt               │ The "diary".                    │
    │                             │ A log of changes that were       │
    │                             │ already made — who did what      │
    │                             │ and when. Helps you catch up     │
    │                             │ if you missed a few days.        │
    ├─────────────────────────────┼──────────────────────────────────┤
    │ Known_Issues.txt            │ The "bug jar".                  │
    │                             │ Stuff that's broken or weird     │
    │                             │ that we know about but haven't   │
    │                             │ fixed yet. Check before you      │
    │                             │ panic — your bug might already   │
    │                             │ be listed here!                  │
    └─────────────────────────────┴──────────────────────────────────┘


====================================================================
    HOW TO USE THIS FOLDER (Simple Rules)
====================================================================

    1. BEFORE you start coding → Read the Plans_and_References.txt
       for your module. This tells you how things should work.

    2. PICK a task → Open Todo.txt and grab a task that isn't
       taken yet. Write your name next to it so others know.

    3. AFTER you finish something → Write a short entry in
       Changelog.txt describing what you did.

    4. IF you find a bug → Add it to Known_Issues.txt with a
       short description. Don't just tell the group chat —
       write it down here so we don't forget.


====================================================================
    ANALOGY: Think of it like a restaurant kitchen 🍳
====================================================================

    Plans_and_References = The Recipe (how the dish should taste)
    Todo                 = The Order Tickets (what dishes to cook)
    Changelog            = The Logbook (what was served today)
    Known_Issues         = The "Out of Stock" board (what's broken)

    The kitchen works because everyone checks the board.
    Our project works the same way — check the files!
