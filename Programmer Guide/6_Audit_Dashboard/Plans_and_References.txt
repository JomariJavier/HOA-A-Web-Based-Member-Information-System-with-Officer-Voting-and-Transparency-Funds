SYSTEM SECURITY & ACCESS AUDITING — PLANS & REFERENCES
====================================================================

REFERENCE: intergrationplan.md -> Phase 4: Financial Auditing (Shared Logic)

Objective: A system-wide audit trail for security and non-repudiation. 
Records "who did what, when, and from where" across all HOA modules.

Tech Stack:
- Backend: Spring Boot, PostgreSQL
- Frontend: React (Vite), Material You Design

Core Features:
- Automatic Harvesting: Captures IP addresses and Browser signatures for all logged actions.
- Centralized Logging: A single service (AuditLogService) handles logging for all other modules.
- Non-Repudiation: Linking actions to authenticated Users.

Core Entities:
- AuditLog: user, action, details, ipAddress, userAgent, createdAt.

Key Views:
- SecurityDashboard.jsx: A specialized view for Superadmins to review system-wide activity.
